01
Ingress
Accept SSE request, verify JWT, establish request/conversation correlation and root trace.
02
Intent + Focus
Classify latest user turn, derive subject/facet from the submitted window, bias uncertain cases toward fetch.
03
Resolve
Turn user references into canonical resource IDs through deterministic resolver/coverage services.
04
Route
Embed query and select manifest-declared agents from the vector index with confidence thresholds.
05
Authorize
Apply structural gates, classification gates, and coverage checks before any agent call.
06
Bind Inputs
Fill agent input schemas from resolved entities and literals. Missing required values drop the agent.
07
Fan-out
Invoke HTTP and MCP agents in parallel with isolated timeouts and failure handling.
08
Synthesize
Treat agent responses as untrusted data; summarize only what came back, then stream deltas.
The LLM proposes references and language. Code resolves IDs, checks access, and decides whether the turn proceeds.

No gateway memory
Conversation context is supplied by the BFF. The gateway re-derives focus per turn.
No model-owned authorization
Chat text never grants access. JWT, policy, and coverage services do.
No fabricated identifiers
The model cannot invent IDs because canonical IDs come only from deterministic resolve.
Gateway alone: a stateless, manifest-driven decisioning runtime between chat clients and specialist systems.
